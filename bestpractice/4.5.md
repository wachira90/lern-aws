# AWS VPC Flow Logs คืออะไร

**AWS VPC Flow Logs** คือ ฟีเจอร์สำหรับบันทึกข้อมูลการจราจร (Traffic) ของเครือข่ายภายใน [Amazon VPC](https://amazon.com) เพื่อให้คุณสามารถดูได้ว่ามีข้อมูลใดวิ่งเข้าหรือออกจากเน็ตเวิร์กของคุณบ้าง โดยมักนำไปใช้ตรวจสอบความปลอดภัยและวิเคราะห์แก้ไขปัญหาเครือข่าย

---

## ข้อมูลที่สามารถบันทึกได้
ข้อมูลการจราจรจะถูกจับในรูปแบบ IP Traffic Header ประกอบไปด้วยรายละเอียดสำคัญ เช่น:
* **IP ต้นทางและปลายทาง:** ที่มาและจุดหมายของการเชื่อมต่อ
* **พอร์ต (Port):** พอร์ตเครือข่ายที่ใช้งาน
* **โปรโตคอล (Protocol):** เช่น TCP, UDP ตามหมายเลข IANA
* **ขนาดข้อมูล:** จำนวนแพ็กเก็ต (Packets) และไบต์ (Bytes) ที่ถูกรับ-ส่ง
* **การดำเนินการ (Action):** สถานะการอนุญาต (`ACCEPT`) หรือถูกปฏิเสธ (`REJECT`) ตามกฎของ Security Group และ Network ACL

---

## ปลายทางในการจัดเก็บข้อมูล (Destination)
บันทึกข้อมูลที่ได้สามารถจัดเก็บและเรียกดูผ่านบริการอื่นๆ ของ AWS ได้แก่:
* **[Amazon CloudWatch](https://amazon.com):** สำหรับตรวจสอบและตั้งค่าแจ้งเตือน (Log Group)
* **[Amazon S3](https://amazon.com):** เหมาะสำหรับจัดเก็บข้อมูลปริมาณมากในระยะยาว

---

## ประโยชน์หลักของการใช้งาน
* **การตรวจสอบความปลอดภัย:** ช่วยตรวจสอบ IP ที่พยายามเชื่อมต่อเข้ามาแบบผิดปกติหรือตรวจสอบทราฟฟิกที่ถูกปฏิเสธ (`REJECT`) เพื่อวิเคราะห์การโจมตี
* **การวิเคราะห์แก้ไขปัญหา (Troubleshooting):** ระบุได้ว่าทำไมทรัพยากร (เช่น [Amazon EC2](https://amazon.com)) ถึงไม่สามารถเชื่อมต่อหากันได้
* **การตรวจสอบการเข้าถึง:** ช่วยให้มองเห็นภาพรวมว่าแอปพลิเคชันของคุณมีการสื่อสารกับปลายทางใดบ้าง

---

## สิ่งที่ Flow Logs ไม่จัดเก็บ
ระบบจะไม่บันทึกทราฟฟิกบางประเภทเพื่อลดขนาดของข้อมูลจัดเก็บ เช่น:
* ทราฟฟิกที่เกิดจาก Instance ไปยัง DNS Server ภายใน (ยกเว้นกรณีที่กำหนดค่าเอง)
* ทราฟฟิกของการเชื่อมต่อ Instance Metadata (`169.254.169.254`)
* ทราฟฟิกของ DHCP และ Windows Time Sync
